Hiệu năng (Performance)
PR1: Trang Board tải xong trong < 2 giây (với 500 task).

PR2: API phản hồi các thao tác CRUD trong < 500 ms.

PR3: Thao tác kéo-thả task trên giao diện phải mượt mà, phản hồi < 200ms.

Khả năng mở rộng & Lưu trữ (Scalability - Đã điều chỉnh)
CR1: Hệ thống sử dụng Docker Volumes để lưu trữ dữ liệu task và file đính kèm bền vững trên ổ cứng máy chủ (Host), không bị mất khi container bị xóa.

CR2: Hệ thống cho phép cấu hình giới hạn tài nguyên (RAM/CPU limit) thông qua file docker-compose.yml.

Độ tin cậy (Reliability - Đã điều chỉnh)
RA1: Dữ liệu task phải đảm bảo tính toàn vẹn (ACID), không bị mất dữ liệu đã lưu nếu container bị tắt đột ngột.

RA32: Container backend phải có cơ chế tự động khởi động lại (Restart Policy: on-failure) nếu gặp lỗi runtime.

Bảo mật (Security)
SR1: Xác thực người dùng bằng email/password (Password phải được mã hóa/hash trong database).

SR2: Phân quyền rõ ràng (Project Manager, Member).

SR3: Chặn truy cập trái phép vào Board không được cấp quyền.

SR4: Code phải xử lý chặn các lỗi bảo mật cơ bản: SQL Injection, XSS (đặc biệt quan trọng dù chạy local).

2.5. Khả năng sử dụng (Usability )
UR1: Giao diện người dùng phải trực quan và dễ sử dụng đối với người mới.

UR2: Người dùng mới phải có thể tạo board và task đầu tiên trong ≤ 10 phút.

UR3: Hệ thống phải hỗ trợ thao tác drag & drop cho việc quản lý task.

UR4: Các thao tác quan trọng phải có thông báo phản hồi rõ ràng cho người dùng.

UR_5: Người dùng phải cài đặt và chạy được toàn bộ hệ thống chỉ với 1 câu lệnh (ví dụ: docker-compose up) trong thời gian < 5 phút.

2.6. Khả năng bảo trì (Maintainability)
MR1: Tổ chức mã nguồn rõ ràng (ví dụ: mô hình MVC hoặc Modular).

MR2: Log lỗi phải được xuất ra Standard Output (stdout) của Docker để dễ dàng debug bằng lệnh docker logs.

2.7. Khả năng tương thích & Tích hợp (Compatibility & Interoperability)
CR1: Chạy tốt trên Chrome/Firefox phiên bản mới nhất.

I1: Cung cấp RESTful API nội bộ để Frontend giao tiếp với Backend.

I2: API trả về đúng HTTP status code (200, 400, 401, 500).
